Architecture Overview

        
  +-----------------+       +-----------------+       +-----------------+
  | Suricata (IDS)  |       | Wazuh Agents    |       | Argus Sentinel  |
  | network traffic |       | endpoint logs   |       | endpoint state  |
  +--------+--------+       +--------+--------+       +--------+--------+
           |  EVE JSON               | alerts.json             | webhook
           +------------+------------+-----------+--------------+
                        |                        |
                  +-----v------------------------v-----+
                  |           shallotd                  |
                  |  normalize > dedup > enrich > AI    |
                  |  SQLite FTS5 | aiohttp dashboard    |
                  +-----+-------------------+----------+
                        |                   |
                  +-----v-----+       +-----v------+
                  | Dashboard |       | Email/SMS  |
                  | port 8844 |       | Alerts     |
                  +-----------+       +------------+
        

        

          shallotd is the central daemon. It ingests alerts from multiple sensors,
          normalizes them, deduplicates, enriches with GeoIP/DNS, and runs AI triage.
          Everything flows into a SQLite database and this dashboard.
        

      


      
      

        
Install Endpoint Agents

        

          Two agent types protect your endpoints. Clove (Wazuh-based) handles log analysis,
          file integrity, and rootkit detection. Argus is the heavy sentinel for credential
          monitoring, session tracking, evidence capture, and anti-tamper. Both work on Windows;
          Linux uses Clove. Pick what fits each machine.

Linux Endpoints

          

            Installs the Clove agent (Wazuh) with custom detection rules.
            Monitors SSH, file integrity, rootkits, reverse shells, suspicious downloads, Docker, and systemd changes.
          

          
One-line install (run as root on the endpoint):

          

            curl -fsSL https://raw.githubusercontent.com/benolenick/security-shallots/main/setup/endpoint/clove \
  | sudo bash -s -- --manager YOUR_SERVER_IP
            
          

          
Options:

          

            
--manager IPWazuh manager IP (required)

            
--name NAMEAgent display name (defaults to hostname)

            
--group GROUPWazuh agent group (default: default)

            
--password PWWazuh registration password

            
--crowdsecAlso install CrowdSec + iptables bouncer

            
--uninstallRemove the agent completely

          

          
Idempotent — safe to re-run on an endpoint that already has it.

        


        
        

          
Windows Endpoints

          

            The Windows installer deploys Argus by default (credential monitoring, Event Logs,
            RDP sessions, USB, evidence capture). Add -Wazuh to also install the
            Clove Wazuh agent for file integrity and log analysis —
            both agents run side-by-side.
          

          
Argus only (run as Administrator in PowerShell):

          

            .\clove.ps1 -Manager YOUR_SERVER_IP
            
          

          
Argus + Clove (Wazuh) together:

          

            .\clove.ps1 -Manager YOUR_SERVER_IP -Wazuh
            
          

          
One-liner (downloads and runs):

          

            irm https://raw.githubusercontent.com/benolenick/security-shallots/main/setup/endpoint/clove.ps1 | iex
            
          

          
Options:

          

            
-Manager IPshallotd server IP (required)

            
-Name NAMEAgent display name (defaults to hostname)

            
-WebhookPort 8855Argus webhook port on server

            
-WebhookSecret "..."Shared secret for Argus auth

            
-WazuhAlso install Wazuh agent (Clove)

            
-WazuhGroup GROUPWazuh agent group

            
-WazuhPassword PWWazuh registration password

            
-UninstallRemove everything

          


          
What each agent monitors on Windows:

          

            

              argus
              Failed logons (4625), account creation (4720), group changes (4728/4732),
                  account lockouts (4740), audit log clearing (1102), credential file access
                  (KeePassXC, SSH keys), RDP sessions, USB devices, evidence capture, anti-tamper
            

            

              clove
              File integrity monitoring, Windows Event Log analysis, rootkit detection,
                  active response, vulnerability detection, security configuration assessment
            

          


          

            Argus uses port 8855 on the server — make sure it’s open.
            The installer auto-installs Python if needed. Both agents run as background services.
          

        

      


      
      

        
Diagnostics — shallot-doctor

        

          shallot-doctor is the all-in-one diagnostic, backup, and repair tool.
          Run it on your central server whenever something seems wrong.
        

        
Full health check:

        

          sudo bash setup/shallot-doctor check
          
        

        
All commands:

        

          
checkFull diagnostic: services, ports, DB integrity, TLS, backups

          
backupBackup config + DB + TLS certs + crontabs

          
restore <file>Restore from a backup tarball

          
fix-tlsRegenerate self-signed TLS certificate

          
fix-cronsReinstall maintenance cron jobs

          
fix-serviceReinstall + restart the systemd service

          
fix-permsFix file permissions and group membership

          
reset-configReset config.yaml to defaults

        

        

          The check command verifies: service status, port availability, database integrity,
          TLS cert validity, backup freshness, and log file accessibility. Start here for any issue.
        

      


      
      

        
Component Guide

        

          

            
suricata Network IDS

            
Inspects all network traffic in real-time using signature-based detection.
            Alerts when packets match known threat patterns (malware C2, scanning, exploits).
            Watches the wire — everything entering and leaving your network.

          

          

            
wazuh Endpoint Security

            
Agent-based monitoring on Linux/Windows endpoints (via Clove installer).
            Covers file integrity, log analysis, rootkit detection, authentication monitoring.
            Alerts when something changes on a host.

          

          

            
argus Endpoint Sentinel

            
Heavy Windows-focused endpoint agent. Monitors Windows Event Logs, credential file access,
            RDP/network sessions, USB devices. Includes evidence capture and anti-tamper.
            The “big brother” for high-value endpoints.

          

          

            
crowdsec Crowd Intelligence

            
Community-powered threat intelligence. Detects brute force, scans, and known bad IPs
            using shared blocklists. Can automatically firewall attackers via bouncers.

          

          

            
pfsense Firewall Logs

            
Ingests pfSense filterlog entries via syslog. Shows blocked/allowed traffic
            at the firewall level — useful for seeing what your firewall is stopping.

          

        

      


      
      

        
Severity Levels & Verdicts

        

          
criticalActive exploitation or compromise. Investigate immediately.

          
highLikely malicious. Needs prompt investigation.

          
mediumSuspicious activity. Review when possible.

          
lowInformational noise. Usually benign.

        

        

          
pendingNot yet triaged. AI reviews in batches, or set manually.

          
investigateNeeds human review. Something looks wrong.

          
escalateConfirmed threat or high-priority finding.

          
suppressKnown noise. Benign scanners, internal traffic, etc.

        

      


      
      

        
Troubleshooting

        

          No alerts appearing
          
    
            
  1. Run sudo bash setup/shallot-doctor check
    2. 
            
  2. Verify Suricata is running: systemctl status suricata
    3. 
            
  3. Check EVE log exists: ls -la /var/log/suricata/eve.json
    4. 
            
  4. Check shallotd logs: journalctl -u shallotd -f | grep -i ingest
    5. 
            
  5. Verify component is enabled in config.yaml under components:
    6. 
          

        

        

          AI triage stuck on “pending”
          
    
            
  1. Check AI config: grep -A5 'ai:' config.yaml
    2. 
            
  2. AI tier must not be none
    3. 
            
  3. For Ollama: curl http://OLLAMA_HOST:11434/api/tags
    4. 
            
  4. Check triage logs: journalctl -u shallotd | grep -i triage
    5. 
          

        

        

          Dashboard not loading
          
    
            
  1. Confirm service: systemctl is-active shallotd
    2. 
            
  2. Confirm port: ss -tlnp | grep 8844
    3. 
            
  3. If HTTPS, you must use https:// and accept the self-signed cert
    4. 
            
  4. Check auth: credentials are in config.yaml under web:
    5. 
          

        

        

          Clove agent not connecting
          
    
            
  1. On endpoint: systemctl status wazuh-agent
    2. 
            
  2. Check manager address: grep '<address>' /var/ossec/etc/ossec.conf
    3. 
            
  3. Test connectivity: nc -z SERVER_IP 1514 and nc -z SERVER_IP 1515
    4. 
            
  4. Check enrollment: cat /var/ossec/etc/client.keys
    5. 
            
  5. Re-run installer (it’s idempotent)
    6. 
          

        

        

          Argus webhook not receiving
          
    
            
  1. Verify port 8855 is open on server firewall
    2. 
            
  2. Test endpoint: curl http://SERVER_IP:8855/api/ingest/argus
    3. 
            
  3. On Windows: python -m argus --config config.toml status
    4. 
            
  4. Check webhook secret matches in both config.toml and server config.yaml
    5. 
          

        

        

          TLS certificate expired
          
    
            
  1. sudo bash setup/shallot-doctor fix-tls
    2. 
            
  2. sudo systemctl restart shallotd
    3. 
          

        

      


      
      

        
What Needs Human Attention

        

          Security Shallots automates ingestion, dedup, enrichment, and AI triage — but these tasks still need a human:
        

        

          
Verdict triageReview “investigate” alerts and decide: suppress or escalate. AI gives recommendations but you make the call.

          
Agent installationDeploy Clove/Argus to new endpoints. Requires SSH or RDP access to each machine.

          
Agent reattachmentWhen an agent goes offline, use the troubleshoot drawer to restart or re-enroll it.

          
Suricata rule tuningAdd custom rules, tune thresholds, suppress noisy signatures via threshold.config.

          
TLS certificate renewalSelf-signed cert expires yearly. Run shallot-doctor fix-tls and restart.

          
Database backupsRun shallot-doctor backup periodically. Verify backups restore cleanly.

          
GeoIP database updatesDownload a fresh GeoLite2-City.mmdb from MaxMind every few months.

          
API key rotationRotate OpenAI/Anthropic/VirusTotal/AbuseIPDB keys in config.yaml periodically.

          troubleshooting steps, and live system status.
